Consonant Tools

Setup guide for the Figma plugin, AI bridge, and accessibility blueline generator.

Prerequisites

  • Figma Desktop required — the browser version doesn't support plugin bridges
  • Node.js v18+ required — nodejs.org
  • Claude Code CLI AI fill — only needed for AI-assisted features 
    npm install -g @anthropic-ai/claude-code
    Sign in with your Claude account (claude login). Works with any Claude Pro or Max plan.

Quick Setup

If you're not comfortable in Terminal, Claude can handle the entire installation for you. Just three things to do first:

  1. Install Node.js from nodejs.org (click the big green button, run the installer)
  2. Install Claude Code — open Terminal and paste: 
    npm install -g @anthropic-ai/claude-code
  3. Sign in — in the same Terminal, run: 
    claude login

Then open Claude Code and type:

claude

Once Claude starts, type this command:

/project:setup

Claude will clone the repo, install dependencies, build the plugin, and install the bridge — then tell you what to do in Figma.

1 Clone the repo

git clone https://github.com/spicyxshrimp/consonant-figma-plugin.git
cd consonant-figma-plugin

2 Install and build

npm install
npm run build

For live rebuilds while developing:

npm run watch

3 Load the plugin in Figma

  1. Open Figma Desktop
  2. Go to Plugins > Development > Import plugin from manifest...
  3. Select the manifest.json file from the cloned repo
  4. The plugin appears as "Consonant Tools" in your plugin menu
You only need to import once. After that, the plugin persists across all files and sessions.

4 Install the figma-console bridge AI fill

The bridge lets Claude Code read and write to your Figma file.

npm install -g figma-console-mcp

The repo's .mcp.json already references it, so Claude Code will auto-start it when you open a session in this directory.

5 Connect the bridge AI fill

  1. Open a Figma file and run the Consonant Tools plugin
  2. Go to the Bridge tab in the plugin
  3. Click Connect
  4. The status should change to Connected with a green dot
The bridge stays connected as long as the plugin window is open. You can switch between tabs without disconnecting.

6 Start Claude Code AI fill

cd consonant-figma-plugin
claude

Claude Code will detect the .mcp.json and start the figma-console MCP server automatically.

Using the A11y Blueline

Select a frame → A11y tab → Check options → Generate Blueline → Copy instruction → Paste in Claude Code

What you can check

Plugin (no bridge needed) Focus Indicators, Focus Order
AI-assisted (bridge required) Heading Hierarchy, Landmarks, Accessible Names, Alt-Text, ARIA Roles, Keyboard Patterns, DOM Strategy, Auto-Rotation
Accessibility Notes (bridge required) VoiceOver, TalkBack, Narrator, React Native, TV Note, General Note

Two modes

  • Generate Blueline — Focus sidebar + cards below the design. Good for a compact overview.
  • Generate Blueline with Sections — One Figma Section per category, each with its own design clone. Good for detailed per-category review.

AI Fill

After generating the scaffolding, the plugin shows an instruction with a Copy button. Click it, then paste into your Claude Code session. Claude will:

  1. Screenshot the design
  2. Read the Figma layer tree
  3. Fill each card with design-specific accessibility specs
AI fill generates a strong first draft (~70% accuracy). A human accessibility engineer should review and refine — especially keyboard interaction models, screen reader announcement strings, and platform-specific behavior.

Team Setup

Every team member runs their own instance. There's no shared server or conflict.

  • Their own Figma Desktop with the plugin loaded
  • Their own Claude Code session signed into their own Claude account
  • The bridge connects locally (localhost) — no ports shared between users

Troubleshooting

Bridge won't connect
Make sure you're using Figma Desktop (not browser). Check that figma-console-mcp is installed: which figma-console-mcp. The plugin window must be open and visible.
AI-assisted checkboxes are grayed out
Connect the bridge first (Bridge tab > Connect). The checkboxes enable once connected.
/project:fill-blueline shows "Unknown skill"
Start Claude Code from inside the consonant-figma-plugin directory. The command file at .claude/commands/fill-blueline.md is loaded at session start — restart if you added it mid-session.
Focus indicators on wrong elements
Detection uses Figma layer names ("Button", "Logo", "Carousel"). Rename layers in Figma for better accuracy.
Cards still showing "Awaiting AI fill..."
The scaffolding was created but AI fill hasn't run. Copy the instruction from the plugin and paste it into Claude Code.
