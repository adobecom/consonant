B: New panel renderer (separate file)

Create a11y-panels.ts that handles panel generation, cloning, and badge/overlay placement. Imports badge helpers from a11y-blueline.ts but owns the panel logic.

Pros

  • Clean separation of concerns
  • a11y-blueline.ts stays focused on cards
  • Easier to maintain independently

Cons

  • More files to manage
  • Some duplication of constants/types

C: Two-pass — flat cards first, then explode

Keep current flat card flow. Add a second "Explode to Panels" button that takes filled cards and expands each into a Section with a cloned design + annotations.

Pros

  • Flat cards still work independently
  • Panels are an optional upgrade
  • User can preview content before committing to heavy clones

Cons

  • Two steps for panels mode
  • More complex data flow
  • Agent nodeId data needed upfront either way

My recommendation: A — the panel infrastructure already exists and works. We're adding overlay + badge logic and fixing grid layout. Click your preference.
