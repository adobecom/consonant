# Fill Blueline Accessibility Annotations

You are filling the accessibility "blueline" annotation cards that were generated by the Consonant Tools Figma plugin. The plugin creates two types of scaffolding:

1. **"Generate Blueline"** — creates detail cards below the design with "Awaiting AI fill..." placeholder text
2. **"Generate Blueline with Sections"** — creates Figma Sections, each with an Instructions card (has category description but no design-specific entries)

Your job is to analyze the design and populate both types with real accessibility specifications.

## Prerequisites

- The figma-console MCP must be connected (you have access to `figma_execute`, `figma_take_screenshot`, etc.)
- The user has already clicked one of the blueline buttons in the plugin

## Steps

### 1. Identify the blueline cards

Use `figma_execute` to find cards that need filling. Check BOTH types:

```js
const page = figma.currentPage;

// Type 1: Detail cards with placeholder text
const placeholders = page.findAll(n =>
  n.type === 'TEXT' && n.characters.includes('Awaiting AI fill')
);
const detailCards = placeholders.map(p => {
  const card = p.parent;
  return { id: card.id, name: card.name, type: 'detail' };
});

// Type 2: Section instruction cards (inside A11y sections, name="Instructions")
const sections = page.findAll(n => n.type === 'SECTION' && n.name.startsWith('A11y '));
const sectionCards = sections.map(s => {
  const card = s.findOne(n => n.type === 'FRAME' && n.name === 'Instructions');
  return card ? { id: card.id, name: s.name.replace('A11y ', ''), type: 'section' } : null;
}).filter(Boolean);

return { detailCards, sectionCards };
```

For section instruction cards, add entries BELOW the existing description text (don't remove the description).

### 2. Screenshot the design frame

Find the original design frame (not a clone). It's typically the selected node, or you can find it by looking for the frame that the blueline annotations reference. Take a screenshot to visually analyze the design.

### 3. Read the design tree

Use `figma_execute` to serialize the design frame's node tree. Get names, types, text content, and hierarchy:

```js
const node = await figma.getNodeByIdAsync('NODE_ID');
function serialize(n, depth) {
  if (depth > 5 || !n.visible) return null;
  const entry = { id: n.id, name: n.name, type: n.type };
  if (n.type === 'TEXT') entry.text = n.characters.slice(0, 200);
  if ('children' in n) {
    entry.children = n.children.map(c => serialize(c, depth + 1)).filter(Boolean);
  }
  return entry;
}
return serialize(node, 0);
```

**Important:** Include `n.id` in the serialization — you'll need node IDs to place category badges on the design in step 5b.

### 4. Generate accessibility content for each category

Based on the design analysis, generate content for each a11y category. Each card's content should follow this format — a descriptive paragraph with structured entries.

**Categories and what to analyze:**

- **Heading Hierarchy**: Identify text elements that should be headings (H1-H6). Look for large/prominent text, section titles. Note: typically one H1 per page.
- **Landmarks**: Identify ARIA landmark regions (banner, main, nav, region). Look for major structural sections.
- **Accessible Names**: Elements needing explicit labels — icon-only buttons, ambiguous links ("Learn more"), unlabeled inputs.
- **Alt-Text**: Images and media. Determine if decorative (alt="") or meaningful (needs description).
- **ARIA Roles & Attributes**: Custom widgets needing ARIA roles/states. Tabs, accordions, carousels, dialogs, etc.
- **Keyboard Patterns**: Expected keyboard behavior for interactive elements. Tab order, arrow key navigation, Enter/Space activation.
- **DOM Strategy**: How visual layout maps to DOM order. Reading order, semantic HTML elements, skip links.
- **Auto-Rotation**: Auto-advancing content (carousels, slideshows). Timing, pause behavior, reduced-motion.
- **VoiceOver**: VoiceOver-specific behavior on iOS/macOS. Rotor actions, swipe gestures, announcement order, platform quirks.
- **TalkBack**: TalkBack-specific behavior on Android. Touch exploration, gesture navigation, platform-specific announcements.
- **Narrator**: Narrator-specific behavior on Windows. Scan mode, landmark navigation, platform quirks.
- **React Native**: React Native accessibility props: accessibilityLabel, accessibilityRole, accessibilityState, accessibilityActions, platform bridging.
- **TV Note**: TV platform accessibility: D-pad navigation, focus management, spatial navigation, remote control interactions.
- **General Note**: Cross-platform accessibility notes. Live regions, announcements, general behavior.

### 5. Fill the cards AND place badges on the design

For each card, use `figma_execute` to:
1. Load the Inter font family (Regular, Bold, Medium, Semi Bold)
2. Find the card by ID
3. Remove the "Awaiting AI fill..." placeholder text node
4. Add new text nodes with the generated content
5. **Place numbered category badges on the design elements** for each entry

Each entry in a card corresponds to a specific element in the design. As you add entries, also place a numbered badge on that element using the `__placeCategoryBadge` global function. The badge number should match the entry order (1, 2, 3...) within that category.

**Category keys for badges:** `headings`, `landmarks`, `names`, `altText`, `aria`, `keyboard`, `dom`, `autoRotation`, `voiceover`, `talkback`, `narrator`, `reactNative`, `tvNote`, `generalNote`

Use this pattern for filling a card:

```js
await figma.loadFontAsync({ family: 'Inter', style: 'Regular' });
await figma.loadFontAsync({ family: 'Inter', style: 'Bold' });
await figma.loadFontAsync({ family: 'Inter', style: 'Semi Bold' });

const card = await figma.getNodeByIdAsync('CARD_ID');
// Remove placeholder
const placeholder = card.findOne(n => n.type === 'TEXT' && n.characters.includes('Awaiting AI fill'));
if (placeholder) placeholder.remove();

// Add content entries
function addEntry(parent, label, value) {
  const row = figma.createFrame();
  row.name = label;
  row.layoutMode = 'VERTICAL';
  row.primaryAxisSizingMode = 'AUTO';
  row.counterAxisSizingMode = 'AUTO';
  row.layoutAlign = 'STRETCH';
  row.clipsContent = false;
  row.itemSpacing = 2;
  row.paddingTop = 6;
  row.paddingBottom = 6;
  row.fills = [];
  row.strokes = [{ type: 'SOLID', color: { r: 0, g: 0, b: 0 }, opacity: 0.1 }];
  row.strokeWeight = 1;
  row.strokeAlign = 'INSIDE';
  row.strokeTopWeight = 0;
  row.strokeRightWeight = 0;
  row.strokeBottomWeight = 1;
  row.strokeLeftWeight = 0;

  const labelNode = figma.createText();
  labelNode.fontName = { family: 'Inter', style: 'Bold' };
  labelNode.fontSize = 13;
  labelNode.characters = label;
  labelNode.opacity = 0.8;
  labelNode.layoutAlign = 'STRETCH';
  labelNode.textAutoResize = 'HEIGHT';
  row.appendChild(labelNode);

  const valueNode = figma.createText();
  valueNode.fontName = { family: 'Inter', style: 'Regular' };
  valueNode.fontSize = 12;
  valueNode.characters = value;
  valueNode.opacity = 0.7;
  valueNode.layoutAlign = 'STRETCH';
  valueNode.textAutoResize = 'HEIGHT';
  row.appendChild(valueNode);

  parent.appendChild(row);
}

// Example entries — replace with actual analyzed content
addEntry(card, 'Hero Title', 'H1 — Primary page heading, always present');
addEntry(card, 'Section Subtitle', 'H2 — Secondary heading under hero');
```

### 5b. Place category badges on the design

For each entry you add to a card, also place a numbered badge on the corresponding element in the design. Use the `__placeCategoryBadge` function:

```js
// Place a badge on the design element for each entry
// Args: targetNodeId, entryIndex (1-based), categoryKey
await globalThis.__placeCategoryBadge('NODE_ID', 1, 'names');
await globalThis.__placeCategoryBadge('NODE_ID', 2, 'names');
```

The badge will appear as a small pill with the number + category icon on top of the element in the design, matching the human-created blueline reference style:
- **Focus Order**: green circle
- **Accessible Names**: navy pill + headphone icon
- **ARIA Roles**: navy pill + `</>` code icon
- **Heading Hierarchy**: navy pill + H icon
- **Landmarks**: light green pill with text
- **General Note / Alt-Text / etc.**: navy pill + category-specific icon

To find the node ID of the element you're annotating, use the design tree you serialized in step 3. Match element names to their node IDs.

### 6. Verify

After filling all cards, take a screenshot of the blueline to verify the content looks correct.

## Content Guidelines

- Be specific to the actual design, not generic a11y advice
- Reference actual element names from the Figma node tree
- Use concise, actionable language — these are specs for developers
- If a category doesn't apply to the design (e.g., no auto-rotating content), write "Not applicable — no auto-advancing content in this component"
- Match the tone of a senior accessibility engineer writing annotations
