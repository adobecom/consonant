Consonant Tools Plugin

Installation guide for designers
What is this plugin? The Consonant Tools plugin connects Figma to the S2A design system. It lets Claude Code read your open Figma file, run design scripts, and produce annotated component specs — all from a chat session. Once installed, you won't need to touch it again.
Fastest option — let Claude do it for you
  1. Open Terminal (press Cmd + Space, type Terminal, press Enter)
  2. Type bash (with a space after it) — but don't press Enter yet
  3. Open the folder you extracted and drag the install.sh file into the Terminal window — this pastes the file path automatically
  4. Press Enter
  5. Open Claude Code and type /setup-consonant-plugin

Claude will download the files, build everything, and guide you through loading the plugin in Figma (that one step has to be done by hand — Claude will tell you exactly what to click). The rest of this guide is for manual installation or troubleshooting.

1

Prerequisites

Make sure you have both tools installed before continuing.

Tool Requirement Where to get it
Figma Desktop Desktop app only — the browser version does not support plugins in development mode figma.com/downloads
Node.js A behind-the-scenes tool that the plugin needs to run. You won't interact with it directly. nodejs.org → click the big green “Download” button, open the file, and follow the installer
How to open Terminal Press Cmd + Space to open Spotlight, type Terminal, and press Return. All commands in this guide are run inside Terminal.
2

Download the plugin files

Run this single command in Terminal. It downloads the repo ZIP, extracts it, and places the files at ~/Desktop/consonant.

curl -L https://github.com/adobecom/consonant/archive/refs/heads/main.zip -o /tmp/consonant.zip \ && unzip -q /tmp/consonant.zip -d /tmp/consonant-extract \ && mv /tmp/consonant-extract/consonant-main ~/Desktop/consonant \ && rm -f /tmp/consonant.zip \ && rm -rf /tmp/consonant-extract \ && echo "Done! Files are at ~/Desktop/consonant"
Want a different install location? Replace every ~/Desktop/consonant in this guide with your preferred path (for example ~/Projects/consonant). Make sure to use the same path consistently across all steps.
3

Build the plugin

Run two build commands in order — one for the plugin UI and one for its MCP server.

This might take a minute Each command will scroll through installation and build output. That is normal. Wait for Terminal to show a prompt again (a $ or % character) before running the next command.
Step 3a — Build the plugin UI
cd ~/Desktop/consonant/apps/consonant-specs-plugin && npm install && node esbuild.config.mjs
Step 3b — Build the specs MCP server
cd ~/Desktop/consonant/apps/consonant-specs-plugin/mcp && npm install && npm run build
4

Load the plugins in Figma

Import the Consonant Tools plugin into Figma — this only needs to be done once.

  1. Open Figma Desktop (not the browser version).
  2. Open any Figma file.
  3. In the top menu go to Plugins → Development → Import plugin from manifest…
  4. Navigate to Desktop → consonant → apps → consonant-specs-plugin
  5. Select manifest.json and click Open.

The plugin should now appear under Plugins → Development.

To open the plugin folder in Finder, run this in Terminal:
open ~/Desktop/consonant/apps/consonant-specs-plugin

5

Verify it works

Two quick checks to confirm everything is wired up correctly.

Check 5a — Plugin UI loads

In Figma, go to Plugins → Development → Consonant Tools. The plugin panel should open and show several tabs (Specs, Bridge, and others).

What success looks like The plugin panel opens without error and the tab bar is visible at the top.
Check 5b — Bridge connection

Start the bridge server in a new Terminal window:

cd ~/Desktop/consonant/apps/consonant-specs-plugin/mcp && node dist/index.js

Then in Figma:

  1. Run Consonant Tools (Plugins → Development → Consonant Tools).
  2. Click the Bridge tab and click Connect.
What success looks like The Bridge tab shows "Connected" and the status dot is green.
Check 5c — Claude can read your Figma file

With the bridge connected, open Claude Code and ask it to read your file. Try typing something like:

Can you see my Figma file? What's in it?

Claude should respond with your file name, page names, and be able to take a screenshot of what you're looking at in Figma.

What success looks like Claude describes your open Figma file by name and shows you a screenshot. This means the full connection — from Claude Code through the bridge to Figma — is working.
6

Updating the plugin

When a new version is released, update by repeating the steps above from Step 2. The manifest path in Figma stays the same — you will not need to re-import it.

Easier option Inside a Claude Code session, run /setup-consonant-plugin. The skill will pull the latest version, rebuild the plugin and MCP server, and confirm success automatically.
